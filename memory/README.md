# Memory Governance

## 1. 定義

本目錄的 Memory 不是聊天逐字稿，而是經過整理、可追溯、可更新的企業知識。

Memory 分為：

- `decisions/`：正式決策與理由
- `lessons/`：錯誤、修正與可重用教訓
- 可選的 `glossary/`：企業術語
- 可選的 `patterns/`：已驗證方法

## 2. 寫入條件

只有符合下列條件才寫入：

- 未來任務會重複使用
- 已確認來源
- 適用範圍明確
- 不包含不必要的個資或機密
- 有 Owner
- 有失效或複查條件

## 3. 禁止事項

不得自動把以下內容當成正式 Memory：

- 單次對話偏好
- 未經核准的決策
- 網路傳聞
- 暫時性價格
- 未完成草案
- 可能識別個人的敏感資料

## 4. 更新原則

新資訊與舊資訊衝突時：

1. 保留舊紀錄
2. 新增 superseded 標記
3. 寫明取代原因
4. 連結新決策
5. 不靜默覆蓋歷史
