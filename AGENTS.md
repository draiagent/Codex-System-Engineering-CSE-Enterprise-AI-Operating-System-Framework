# AGENTS.md — Enterprise Codex Operating Rules

## 1. Mission

本儲存庫的任務是將企業知識、部門 SOP、AI Skills、專責 Agents、跨部門 Workflows 與治理規則，轉化為可版本控制、可測試、可審核的 Codex 工作系統。

Codex 在本儲存庫工作時，優先追求：

1. 正確性
2. 可追溯性
3. 可重複性
4. 最小必要變更
5. 資安與法遵
6. 可維護性
7. 執行效率

## 2. Instruction Priority

遇到衝突時，依下列順序處理：

1. 使用者本次明確要求
2. 系統與平台安全規則
3. 距離目標檔案最近的 `AGENTS.md`
4. 本根目錄 `AGENTS.md`
5. `context/` 內的企業與部門規範
6. Skill 或 Workflow 文件
7. 一般慣例

若仍無法判定，不得自行猜測重要商業事實；應提出最小必要問題或清楚標記假設。

## 3. Required Operating Loop

任何超過三個步驟的任務，依序執行：

```text
Understand
→ Inspect
→ Plan
→ Execute
→ Validate
→ Report
```

### 3.1 Understand

先確認：

- 任務目標
- 預期輸出
- 使用對象
- 時間範圍
- 風險等級
- 不可變動項目

### 3.2 Inspect

修改前先讀取：

- 相關檔案
- 鄰近的 `AGENTS.md`
- 對應 Skill
- 對應 Workflow
- 測試或驗收規格
- 既有命名與格式

不得在未讀取現有內容前，大幅覆寫檔案。

### 3.3 Plan

符合任一條件時，先列計畫：

- 涉及 3 個以上檔案
- 涉及跨部門工作流
- 變更資料結構
- 涉及權限、資安或法規
- 需求存在多種合理解法
- 可能破壞既有相容性

### 3.4 Execute

執行時：

- 優先最小差異
- 不修改無關檔案
- 不虛構資料、測試結果、引用或核准狀態
- 不刪除仍被引用的規範
- 不將機密寫入公開檔案
- 不將短期任務資訊寫入永久規則

### 3.5 Validate

至少檢查：

- Markdown 結構
- 內部連結
- 命名一致性
- 輸入／輸出是否完整
- 是否符合 Definition of Done
- 是否有未標示假設
- 是否有敏感資訊
- 是否需要人類核准

### 3.6 Report

完成後回報：

- 變更摘要
- 主要檔案
- 驗證方式
- 尚未解決事項
- 風險或人工核准點

不得聲稱未實際執行的測試已通過。

## 4. Repository Boundaries

### 4.1 `context/`

放置長期有效的企業或部門脈絡。

禁止放置：

- 單次活動資料
- 暫時性工作清單
- 未確認的傳聞
- 密碼、Token、私鑰
- 大量未整理原始資料

### 4.2 `memory/`

只保存可追溯的決策與教訓。

每一筆記錄必須包含：

- 日期
- 來源
- 負責人
- 決策或教訓
- 適用範圍
- 失效條件

### 4.3 `skills/`

每個 Skill：

- 只解決一個明確工作
- 必須有清楚 description
- 必須定義輸入、輸出、步驟、品質檢查
- 必須列出何時不應使用
- 優先包含 2–3 個具體使用案例
- 不得依賴未說明的隱性上下文

### 4.4 `agents/`

Agent 是角色規格，不是人格故事。

每個 Agent 必須定義：

- 單一責任
- 可接受輸入
- 必須輸出
- 可使用來源
- 禁止事項
- 交接對象
- 完成條件

### 4.5 `workflows/`

Workflow 必須包含：

- 觸發條件
- 步驟順序
- 每步負責角色
- 輸入與輸出
- 品質閘門
- 失敗處理
- 停止條件
- 人工核准點

## 5. Writing Standards

- 使用繁體中文為主；必要技術名詞保留英文。
- 標題採句意清楚的名詞或動作。
- 一段只處理一個概念。
- 優先使用表格表達比較，使用流程圖表達順序。
- 不使用空泛詞語，例如「全面賦能」「顛覆式提升」，除非有指標。
- 所有強制規範使用「必須／不得」。
- 建議使用「建議／優先」。
- 不確定內容使用「假設／待確認」。

## 6. Security and Privacy

不得提交：

- API Key
- Access Token
- 密碼
- 私鑰
- 身分證號
- 完整健康紀錄
- 未去識別客戶資料
- 未公開財務資料
- 受 NDA 約束的原文

發現疑似機密時：

1. 停止複製或擴散
2. 以 `<REDACTED>` 取代
3. 指出風險
4. 建議移至 Secret Manager 或受控文件系統

## 7. High-Risk Domains

下列任務僅能產生草案或分析，不得宣稱為最終專業意見：

- 法律
- 醫療
- 財務投資
- 食品與健康宣稱
- 人事處分
- 品質放行
- 合約承諾
- 法規合規判定

輸出必須標記：

- 資料來源
- 假設
- 不確定性
- 人類審核點

## 8. Change Rules

### 可以直接修改

- 拼字與格式
- 斷裂連結
- 明顯重複段落
- 不改變語意的結構改善

### 必須先提出計畫

- 修改企業核心規則
- 改變 Skill 的輸入或輸出契約
- 改變 Agent 權限
- 刪除治理條款
- 改變高風險流程
- 大規模重新命名

## 9. Definition of Done

工作完成至少符合：

- 任務要求已覆蓋
- 無已知重大錯誤
- 相關連結有效
- 格式一致
- 來源與假設已標明
- 高風險內容已有人工核准點
- 沒有新增機密資訊
- 已記錄驗證結果
